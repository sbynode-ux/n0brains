// about

What it gives your agent.

n0brains-signals is the agent-facing wrapper around the n0brains Signals API. It teaches any LLM agent — Claude Code, OpenClaw, custom orchestrators — how to authenticate, call the five live-data endpoints, and reason about the responses without hallucinating numbers.

Drop it in, set NB_API_KEY, and your agent can answer market-aware questions on the next prompt. No SDK to wire up, no schemas to maintain. Just markdown the model already knows how to read.

// the 5 tools

GET /signals free + pro
get_signals

List active signals, filterable by asset, type, direction, urgency. The default question — start here 90% of the time.

GET /signals/{id} free + pro
get_signal

Fetch a single signal by ID. Use when following up on a webhook alert or a previous reference.

GET /levels/{coin} free + pro
get_levels

Nearest support / resistance from the Hyperliquid order book. Use for sizing entries and stops.

GET /market-opens free + pro
get_market_opens

BTC / ETH / SOL opening prices at Tokyo, Shanghai, HK, London, NY. Use for session-bias trades.

WS wss://api.n0brains.com/stream pro only
stream_signals

Real-time WebSocket feed. Every new signal arrives in milliseconds. Pair with REST backfill on reconnect — see examples/stream-and-react.md.

// 30-second example

Three lines to your first signal.

# Ask the agent: "find a high-urgency BTC whale move"
from n0brains import get_signals
sigs = get_signals(asset="BTC", signal_type="whale", urgency="high")
for s in sigs["signals"]:
    print(f"{s['direction']:>8}  score={s['score']:.2f}  {s['summary']}")
